Intervals and scheduling variations

Intervals

Interval problems fall under 3 basic construct:

  • 1) Sort by start time: Sorting by start time is a great approach in problems where we want to find the overlaps. For example inserting an interval in an already sorted list of intervals, merging overlapping intervals etc. In such problems, sorting by end time does not add any value to the solution we are aiming for.
  • 2) Sort by end time: Sorting by end time is usually effective in problems where we want to find an optimized or min/max solution. For example remove min intervals to make list of intervals non-overlapping. In this problem we want to keep the intervals with smallest end time because that greedily ensures that there are fewer and subsequently no overlaps.
  • 3) Sometime we would want to take the start and end time apart and sort them individually. This method comes handy in cases when we want to determine maximum intervals that overlap in the list.

Job Scheduling

There are 3 common types of job scheduling problems listed below:

  • 1) End time by which the job should be done and the profit for each
  • 2) Start and end or duration of each job and the profits from each. Jobs need to be scheduled without overlap.
  • 3) Start and end time ( here profit is assumned to be one for all). This is basically an interval problem from sequence number 2 above.
